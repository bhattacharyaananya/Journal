#01

Blue-Team & SOC Engineering

Detection engineering, incident response, and threat hunting.

This domain covers the defensive backbone of modern operations: how telemetry flows into SIEMs, how detections are authored, and how incidents are worked end-to-end inside a SOC.

  • Designing and tuning SIEM pipelines with Wazuh, Splunk, and related tooling.
  • Authoring detection content: rules, correlation searches, dashboards, and alerts.
  • Threat hunting workflows, from hypothesis creation to investigation closure.
  • Incident response runbooks and case studies based on lab-built intrusions.

#02

Offensive Security & VAPT

Red teaming, exploitation, and active defense.

This track focuses on adversarial techniques: how attackers perform reconnaissance, exploit weaknesses, move laterally, and how those actions appear from the defender’s perspective.

  • End-to-end VAPT write-ups, from scoping and recon to post-exploitation.
  • Custom exploit scripts and proof-of-concept payloads, shared responsibly.
  • Honeypots, deception infrastructure, and active defense experiments.
  • Active Directory attack paths, misconfigurations, and lateral movement scenarios.

#03

Digital Forensics & Incident Response

Memory, disk, network, and blockchain forensics.

Digital forensics within ResearchOrbit is about reconstruction: taking artifacts from compromised environments and turning them into a coherent, defensible incident timeline.

  • Host artifact analysis: registry hives, browser traces, event logs, and OS artifacts.
  • Memory and disk forensics workflows using common toolchains.
  • Network forensics from raw .pcap captures to narrative timelines.
  • Blockchain ledger tracking and transaction analysis for fraud and abuse investigations.

#04

Research, Cryptography & Emerging Security

Crypto flaws, reverse engineering, and experimental work.

ResearchOrbit also serves as a notebook for lower-level and emerging topics, where ideas are explored before they harden into production-ready techniques.

  • Explanations of cryptographic flaws, misuse patterns, and implementation pitfalls.
  • Reverse engineering binaries, protocols, and obfuscated logic.
  • Explorations of post-quantum cryptography and hardware-assisted security.
  • Experimental work in ML-driven detection and blockchain security models.

How to read ResearchOrbit

Each article aims to be reproducible: where possible, it includes lab environment details, tooling, and step-by-step procedures. Offensive techniques are presented for educational and defensive purposes; only run them in isolated, authorized environments.

For the source and automation that power this journal, refer to the repository at github.com/bhattacharyaananya/Journal.
